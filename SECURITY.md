# Security Policy

## Supported Versions
Security fixes are provided for the latest code on the `main` branch.

## Reporting a Vulnerability
Please do not report security issues in public GitHub issues.

Use one of these private channels:
1. GitHub "Report a vulnerability" (preferred) in the repository Security tab.
2. Private contact with the maintainer via GitHub profile: https://github.com/khnychenkoav

When reporting, include:
- affected component/module;
- reproduction steps;
- potential impact;
- any proof-of-concept details;
- your suggested remediation (if available).

## Response Process
- Initial acknowledgment target: within 72 hours.
- We will validate, triage severity, and prepare a fix.
- Coordinated disclosure is preferred after a patch is available.
